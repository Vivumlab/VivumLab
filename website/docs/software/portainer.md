## SETUP

### Enabling portainer

#### Command:

**`vlab set portainer.enable True`**

#### File alteration:

set the appropriate service settings in `settings/config.yml` to true

eg.
```
portainer
  enable: True
```

#### Finalising changes:

run: **`vlab update_one service=portainer`**

## FIRST RUN

!!! **DEVELOPERS**: make sure that you include any information that the user requires to get started, below. !!! <br>

#### ADMINISTRATOR SETUP

Navigate to *https://{{ portainer.domain }}/admin*

Create an account with your desired username; as this is the first user, portainer makes this account the administrator.

## OPTIONS

### HTTPS_ONLY
*Default: False* <br>
*Options: True/False*

#### Command:

**`vlab set portainer.https_only True`**

#### File alteration:

set the appropriate service settings in `settings/config.yml` to true

eg.
```
portainer
  https_only: True
```

##### Finalising changes:

run: **`vlab update_one service=portainer`**

### AUTH
*Default: False* <br>
*Options: True/False*

#### Command:

### DOMAIN
*Default: False* <br>
*NOTE: include the sitename and top level domain suffix. eg. name.com, site.net*

#### Command:

#### File alteration:

set the appropriate service settings in `settings/config.yml` to true

### SUBDOMAIN
*Default: portainer* <br>
*NOTE: Periods/ delimiters are not required. eg. 'media' will set the full URL as 'media.{{domain}}'*

set the appropriate service settings in `settings/config.yml` to true
